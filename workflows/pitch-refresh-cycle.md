# Pitch Refresh Cycle

## Objective

Refresh the pitch only after strategy and claims are stable enough.

## Sequence

1. Review `docs/process/claims-register.md`
2. Remove or downgrade any unsupported pitch claims
3. Pull validated conclusions from `docs/venture/business-plan.md`
4. Rewrite `docs/venture/pitch.md` for clarity and memorability
5. Run the brand agent for wording discipline
6. Update the claims register if new wording introduces new claims

## Rule

The pitch should never be more certain than the research base.
