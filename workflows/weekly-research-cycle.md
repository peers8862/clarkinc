# Weekly Research Cycle

## Objective

Run a disciplined weekly cycle that turns open questions into validated or rejected claims.

## Monday

- Review `docs/research-board.md`
- Confirm the week's top priorities
- Assign tasks to the research agent

## Tuesday

- Gather evidence
- Populate `docs/research.md`
- Use templates in `templates/` for consistency

## Wednesday

- Synthesize findings
- Mark each workstream as validated, rejected, or still too weak to use

## Thursday

- Hand validated findings to the business-plan agent or orchestrator
- Update `docs/decision-log.md` and `docs/claims-register.md` if needed

## Friday

- Promote validated conclusions into the core docs
- Record what still cannot be claimed
