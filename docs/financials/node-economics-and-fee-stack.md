# Node Economics And Fee Stack

## Purpose

This document converts CLARK's revenue logic into an explicit first-pass investor model.

It is a planning model, not an audited forecast.

## Economic Sequence

CLARK's first proof sequence should be:

1. training contribution proves real revenue and operational discipline
2. one partner-facility onboarding proves that CLARK can charge for node development
3. recurring node support proves platform value beyond one-time setup
4. equity remains upside, not the justification for weak service economics

## Fee Stack

### 1. Direct Training Contribution

Use the training base case as the first proof point:

- Year 1 base-case gross revenue: `CAD 211,200`
- Year 1 base-case net CLARK contribution after direct instructional costs and Niagara participation: `CAD 62,536`

This is the first commercial proof that the model can produce real revenue before broader network economics mature.

### 2. Discovery And Qualification Fees

Working band:

- `CAD 2,500 to CAD 7,500` per serious facility engagement

### 3. Node Onboarding Fees

Working band:

- `CAD 15,000 to CAD 40,000` per node

### 4. Recurring Node Enablement Fees

Working band:

- `CAD 2,000 to CAD 8,000` per month per active node

### 5. Software Monetization

- bundled by default into Assembly Centre / Assembly Center participation
- separately billable only for non-center or limited-access users

### 6. Ownership Economics

- default ownership target: `10%`
- practical upper band: `15% to 20%`

Equity should be treated as long-term upside and optional capital recycling, not as the basis for early revenue claims.

## First Proof-Case Model

An investor-ready first proof case can be explained as:

- base-case Hamilton training contribution: about `CAD 62,536`
- one paid discovery engagement: about `CAD 5,000`
- one onboarding engagement: about `CAD 25,000`
- one active node on recurring enablement for 12 months at `CAD 3,500` per month: about `CAD 42,000`

Illustrative first proof-case revenue contribution:

- about `CAD 134,536`

This is not a scale case. It is a proof case showing that CLARK can generate revenue from both direct operation and network enablement before mature platform scale.

## 24-Month Proof Logic

By 24 months, CLARK should aim to prove:

- repeatable training contribution
- at least one paid node onboarding path
- at least one recurring node-support relationship
- software value visible enough to justify bundling

## Rules

- Do not count equity appreciation as operating revenue.
- Do not waive onboarding economics casually.
- Do not let recurring node support become unpaid bespoke consulting.
- Do not describe logistics fees as meaningful before real corridor volume exists.
